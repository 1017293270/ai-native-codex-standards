# Security Policy

## Supported Versions

This repository is a standards and documentation project. Security fixes apply to the latest version
on the `main` branch.

## Reporting a Vulnerability

Please do not open public issues for sensitive vulnerabilities such as leaked credentials,
private URLs, or unsafe installation behavior.

Instead, report privately to the repository owner through GitHub.

## Security Expectations

Contributions must not include:

- Real API keys, tokens, passwords, or secrets.
- Private company URLs or customer data.
- Machine-specific absolute paths.
- Unsafe install commands that overwrite unrelated files.
- Rules that encourage bypassing authorization, validation, auditing, or high-risk confirmations.
